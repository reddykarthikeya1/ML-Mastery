## 📝 How to Take Notes

### For Each Topic:
1. Navigate to the appropriate phase folder
2. Create a new file: `Topic-Name.md`
3. Use the template below

### Note Template
```markdown
# Topic Name

## 🎯 Key Concepts
- Concept 1
- Concept 2

## 📚 Theory
Write your explanation here...

## 💻 Code Examples
```python
# Your code here
def example():
    pass
```

## 🖼 Diagrams/Images
![Description](images/diagram.png)

## 🔗 Related Topics
- [[Linked Topic 1]]
- [[Linked Topic 2]]

## ❓ Practice Questions
1. Question here?

## 📌 Summary
Key takeaways from this topic...
```

## 🖼 Adding Images

### Method 1: Paste Screenshot
1. Take screenshot (Win+Shift+S)
2. Place cursor where you want image
3. Press `Ctrl+Alt+V` (with Paste Image extension)

### Method 2: Drag & Drop
1. Save image to `images/` folder
2. Use: `![Description](images/filename.png)`

## 🔗 Linking Between Notes

Use double brackets to link notes:
```markdown
[[Linear-Algebra-Basics]]
[[Python-Functions]]
```

## ✅ Best Practices

1. **One topic per file** - Keep notes focused
2. **Use consistent naming** - `Topic-Subtopic.md`
3. **Add dates** - When you studied something
4. **Include code** - Always add working examples
5. **Draw diagrams** - Visual memory helps retention
6. **Link related topics** - Build your knowledge graph
7. **Review regularly** - Use spaced repetition

## 📅 Study Workflow

1. **Before Learning**
   - Check roadmap for topic
   - Create note file with template

2. **During Learning**
   - Take notes in real-time
   - Add code snippets
   - Screenshot important diagrams

3. **After Learning**
   - Write summary
   - Add practice questions
   - Link to related topics
   - Commit to git (if using)
